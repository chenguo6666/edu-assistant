# 项目需求与设计文档（PRD）

> 本文档为项目开发前的需求与设计基准文档。开发过程中若发现问题或设计偏差，须直接提出，经用户确认后方可修改。

---

## 一、项目概述

* **项目名称**：EduAssistant 教育助手 AI Agent
* **项目背景**：大四课程项目，要求开发一个教育领域的 AI Agent Web 应用，展示 Agent 的任务分解、工具调用等核心能力。
* **目标用户**：在校大学生，主要面向有学习辅导需求和 CS 保研信息查询需求的学生群体。
* **核心价值**：通过 AI Agent 自动分解复杂教育任务、调用多种工具完成学习辅导，同时提供 CS 保研信息的智能问答服务。

---

## 二、目标与成功指标

* **主要目标**：
  1. 展示 AI Agent 的完整工作流程（任务分解 → 工具调用 → 结果整合）
  2. 实现通用教育辅导功能（总结、出题、知识点提取、学习计划）
  3. 实现 CS 保研信息的 RAG 智能问答
  4. 提供类 ChatGPT 的流畅交互体验

* **成功指标**（可量化）：
  * [x] 指标 1：Agent 能正确分解复合任务并依次调用工具完成
  * [x] 指标 2：前端能实时展示 Agent 的思考过程和工具调用步骤
  * [x] 指标 3：RAG 问答能基于上传文档 / 预置保研数据给出相关回答
  * [x] 指标 4：联网搜索工具能返回实时信息辅助回答

## 三、功能需求

### 3.1 核心功能

| 功能 | 描述 | 优先级 |
|-|-|-|
| 用户注册/登录 | 基于 JWT 的简单账户体系 | `必须实现` |
| 用户中心 | 填写年级、专业、感兴趣学科等个人信息，用于个性化回答 | `必须实现` |
| 聊天对话 | 类 ChatGPT 界面，支持多轮对话、对话历史管理 | `必须实现` |
| Agent 工作流展示 | 前端实时展示任务分解步骤、工具调用过程和结果 | `必须实现` |
| 通用教育 Agent | 文本总结、习题生成、知识点提取、学习计划生成 4 个工具 | `必须实现` |
| CS 保研信息 Agent | 院校信息查询、保研时间线、条件匹配推荐 3 个工具 | `必须实现` |
| 模式切换 | 顶部 Tab 在「教育助手」和「保研助手」之间切换 | `必须实现` |
| 联网搜索工具 | 通过 SerpAPI 实时搜索互联网信息，两个 Agent 均可调用 | `必须实现` |
| 文档上传 RAG | 用户上传文档建立向量索引，支持基于文档的问答 | `必须实现` |
| 预置保研知识库 | 手动收集的 CS 保研数据（10-15 所高校），存入向量库 | `必须实现` |

### 3.2 扩展功能

| 功能 | 描述 | 优先级 |
|-|-|-|
| 对话标题自动生成 | 根据首条消息自动生成对话标题 | `期望实现` |

## 四、非功能需求

* **性能**：单次 Agent 执行应在 30 秒内完成（受限于 LLM API 响应速度）
* **兼容性**：支持 Chrome / Edge 等主流浏览器
* **安全性**：密码 bcrypt 加密存储；API Key 不暴露到前端；JWT 认证保护接口
* **其他**：本地部署运行，无需云服务器

## 五、技术选型

| 类别 | 选型 | 备注 |
|-|-|-|
| 前端框架 | Vue 3 + Vite + TypeScript | SPA 应用 |
| UI 组件库 | Naive UI | 轻量现代，TS 支持好 |
| 状态管理 | Pinia | Vue 3 官方推荐 |
| 后端框架 | FastAPI (Python) | 与 LangChain 同生态，异步支持好 |
| AI 框架 | LangChain | Agent 构建、工具定义、RAG |
| 大模型 | 硅基流动 Qwen3.5 | OpenAI 兼容接口，`Qwen/Qwen3.5-397B-A17B` |
| Embedding | 硅基流动 Embedding 模型 | 文档向量化 |
| 向量数据库 | ChromaDB | 轻量本地，零配置 |
| 关系数据库 | SQLite + SQLAlchemy | 课程项目够用 |
| 认证方案 | JWT (python-jose + bcrypt) | 简单可靠 |
| 联网搜索 | SerpAPI | LangChain 内置集成，有免费额度 |

## 六、数据设计

### 6.1 关系数据库（SQLite）

**users 表**

| 字段 | 类型 | 说明 |
|-|-|-|
| id | INTEGER PK | 自增主键 |
| username | VARCHAR(50) UNIQUE | 用户名 |
| password_hash | VARCHAR(255) | bcrypt 哈希密码 |
| grade | VARCHAR(20) NULL | 年级（如「大三」） |
| major | VARCHAR(100) NULL | 专业 |
| subjects | TEXT NULL | 感兴趣学科（JSON 数组） |
| gpa | FLOAT NULL | GPA（保研模块用） |
| created_at | DATETIME | 注册时间 |
| updated_at | DATETIME | 更新时间 |

**conversations 表**

| 字段 | 类型 | 说明 |
|-|-|-|
| id | INTEGER PK | 自增主键 |
| user_id | INTEGER FK | 关联用户 |
| title | VARCHAR(200) | 对话标题 |
| mode | VARCHAR(20) | 模式：`edu` / `admission` |
| created_at | DATETIME | 创建时间 |
| updated_at | DATETIME | 最后活跃时间 |

**messages 表**

| 字段 | 类型 | 说明 |
|-|-|-|
| id | INTEGER PK | 自增主键 |
| conversation_id | INTEGER FK | 关联对话 |
| role | VARCHAR(10) | `user` / `assistant` |
| content | TEXT | 消息内容 |
| agent_steps | TEXT NULL | Agent 中间步骤（JSON） |
| created_at | DATETIME | 发送时间 |

**documents 表**

| 字段 | 类型 | 说明 |
|-|-|-|
| id | INTEGER PK | 自增主键 |
| user_id | INTEGER FK | 上传者 |
| filename | VARCHAR(255) | 原始文件名 |
| file_path | VARCHAR(500) | 存储路径 |
| file_type | VARCHAR(20) | 文件类型 |
| chunk_count | INTEGER | 分块数量 |
| collection_name | VARCHAR(100) | ChromaDB Collection 名 |
| created_at | DATETIME | 上传时间 |

### 6.2 向量数据库（ChromaDB）

| Collection | 用途 | 数据来源 |
|-|-|-|
| `user_{user_id}_docs` | 用户个人文档 | 用户上传 |
| `admission_knowledge` | CS 保研知识库 | 预置数据（手动收集） |

## 七、界面与交互设计

* **主界面**：类 ChatGPT 布局
  * 顶部：Logo + 模式切换 Tab（教育助手 / 保研助手） + 用户头像
  * 左侧：对话历史列表 + 新建对话按钮
  * 右侧：聊天消息区域 + 底部输入框

```
┌─────────────────────────────────────────────────┐
│  Logo    [教育助手] [保研助手]      [用户头像]    │
├──────────┬──────────────────────────────────────┤
│ + 新建对话│                                      │
│──────────│      消息气泡区域                      │
│ 对话1    │  ┌───────────────────────────────┐    │
│ 对话2    │  │ 用户: 总结课文并生成选择题      │    │
│ 对话3    │  │ AI: [Agent步骤展示-可折叠]     │    │
│          │  │     ▶ 步骤1: 调用总结工具      │    │
│          │  │     ▶ 步骤2: 调用习题生成工具  │    │
│          │  │   最终回答（Markdown渲染）      │    │
│          │  └───────────────────────────────┘    │
│          │  ┌───────────────────────────────┐    │
│          │  │ [上传文件] [输入消息...]  [发送]│    │
│          │  └───────────────────────────────┘    │
└──────────┴──────────────────────────────────────┘
```

* **核心交互流程**：
  1. 用户在输入框输入教育任务 → 发送
  2. 前端通过 WebSocket 连接后端
  3. 后端 Agent 开始执行，实时推送中间步骤
  4. 前端展示：思考中 → 工具调用卡片（逐步出现） → 流式最终回答
  5. 完成后消息存入历史

## 八、约束与边界

* 本项目**不包含**：
  * 多用户权限管理（无管理员角色）
  * 付费功能或额度限制
  * 移动端适配
  * 部署到公网服务器
* 已知限制：
  * LLM 响应速度取决于硅基流动平台的 API 性能
  * 保研数据为手动收集，覆盖范围有限（10-15 所高校 CS 专业）
  * 联网搜索依赖 SerpAPI 免费额度（每月 100 次）
